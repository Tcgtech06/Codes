## Request Flow

### Public Request (Get Companies)
```
1. Frontend â†’ GET /api/v1/companies
2. Echo Router â†’ getCompanies handler
3. CORS Middleware â†’ Allow request
4. Logger Middleware â†’ Log request
5. Handler â†’ companyRepo.GetAll()
6. Repository â†’ SQL Query to Supabase
7. Supabase â†’ Return data
8. Repository â†’ Map to Company models
9. Handler â†’ JSON response
10. Frontend â†’ Display companies
```

### Protected Request (Create Company)
```
1. Frontend â†’ POST /api/v1/companies + JWT token
2. Echo Router â†’ createCompany handler
3. CORS Middleware â†’ Allow request
4. Logger Middleware â†’ Log request
5. JWT Middleware â†’ Verify token
   â”œâ”€ Valid â†’ Continue
   â””â”€ Invalid â†’ Return 401 Unauthorized
6. Handler â†’ companyRepo.Create()
7. Repository â†’ Generate UUID
8. Repository â†’ INSERT query to Supabase
9. Supabase â†’ Save data, return success
10. Repository â†’ Return created company
11. Handler â†’ JSON response
12. Frontend â†’ Show success message
```

## Key Design Patterns

1. **Repository Pattern**
   - Separates data access logic
   - Easy to test and maintain
   - Can swap database easily

2. **Dependency Injection**
   - Repositories injected into handlers
   - Loose coupling
   - Better testability

3. **Middleware Chain**
   - CORS â†’ Logger â†’ JWT â†’ Handler
   - Reusable components
   - Clean separation of concerns

4. **RESTful API**
   - Standard HTTP methods
   - Resource-based URLs
   - JSON responses

5. **Environment Configuration**
   - .env file for settings
   - Easy to change per environment
   - Secrets not in code
